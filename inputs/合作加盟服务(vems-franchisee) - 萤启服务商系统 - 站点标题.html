    3. 核心业务

    3.1 售货机管理

    3.1.1 描述

    售货机管理是合作加盟服务的核心功能，负责配置售货机创建、分配、运营到最终撤机的全生命周期进行管理

    • 售货机生命周期管理: 支持售货机的创建、信息修改、分配、撤机。
    • 运营状态管理: 控制售货机的启用、停用、激活等状态转换。
    • 数据查询: 提供售货机的列表、详情及生命周期变更记录的查询。

    1. 监听队列queue.franchisee
    2. 解析并校验消息，vemId是否为空
    3. 请求售货机接口根据售货机Id获取售货机详情
    4. 根据售货机服务商Id构造routingkey
    5. 发送激活同步消息，exchange&routingkey：VEMISSUE.topic/routingkey.syncVemNature.{服务商id}

    MQ消息体

    {
    "issueId": "ISSUE2023121500016",
    "configType": "syncVemNature",
    "vemNatureCreateCommand": {
        "vemId": "VEM0001", //售货机Id
        "vemNatureName": "自营",// 售货机性质名称
        "updateTime": "2025-12-12 16:39:52"
    }
}
    1. 修改售货机性质

    1. 参数校验
    2. 校验商户是否合作中
    3. 调用售货机服务修改售货机信息
    4. feign调用成功，修改售货机绑定城市
    5. 发送激活同步消息，exchange&routingkey：VEMISSUE.topic/routingkey.syncVemNature.{服务商id}

    MQ消息体

    {
    "issueId": "ISSUE2023121500016",
    "configType": "syncVemNature",
    "vemNatureCreateCommand": {
        "vemId": "VEM0001", //售货机Id
        "vemNatureName": "自营",// 售货机性质名称
        "updateTime": "2025-12-12 16:39:52"
    }
}
    1. 监听售货机库存盘点记录

    1. 监听盘点消息，队列：queue.franchisee，路由Key：inventory.result
    2. 解析消息，判断tenantId是否为空
    3. 保存盘点记录，根据tenantId获取对应服务商Id
    4. 根据服务商Id构造routingkey
    5. 发送库存盘点同步消息，exchange&routingKey:VEMISSUE.topic/routingkey.syncVemInventoryRecord.{服务商Id}

    3.9 萤启会员专享价配置

    3.9.1 流程图

    1.活动配置和导入售货机

    用于完成活动基础信息、商品规则配置，并通过 Excel 导入关联售货机
    导入采用 EasyExcel 流式解析，避免一次性加载导致内存峰值与超时。

    1.1 校验规则

    • 商品规则互斥：
      • 同一 merchandise_id 在同一活动只允许一条规则（DB 唯一约束activity_id+merchandise_id）
      • pricing_type=1（一口价） → fixed_price 必填；pricing_type=2（折扣比例） → discount_rate 必填（互斥）
      • limit_group：1限购组 / 2不限购组互斥
    • 限购共享次数：
      • 若存在任一限购组商品（limit_group=1），则 limited_shared_count > 0
      • 限购组为空允许 limited_shared_count 为空
    • 导入文件格式：仅 xls/xlsx
    • 导入状态限制：仅 status=未发放 允许导入

    1.2 导入过滤与落表规则

    • 对每台售货机执行过滤：
      • 合作中
      • 未撤机、未停用
      • 微超类型售货机，校验支付方式是否包含会员扫码付
    • 落表策略：
      • 符合：valid_flag=1
      • 不符合：valid_flag=0，写 invalid_reason
    • 去重策略：
      • (activity_id, machine_id) 唯一；

    2.配置下发

    2.1 规则与限制

    • 状态限制：仅 未发放(1) 允许下发
    • 配置完整性：必须已配置商品规则且存在有效售货机（valid_flag=1）

    2.2 配置下发

    • 正式下发的唯一入口：手动立即下发
    • 下发后是否生效由终端控制
    • 服务商商品Id转运营商商品Id：下发时，商品Id需要转成运营商商品

    3.手动中止/到期

    3.1 规则与限制

    • 人工中止优先级高于自动到期：一旦中止，活动进入终态（已中止）
    • 仅 status=已发放(2) 允许中止
    • 中止后活动为终态：不可再下发、不可恢复、不可修改
    • 终止动作由 CAS 控制：已发放(2)→已中止(3) 只会成功一次
    • 中止和定时任务到期都通过 CAS 抢占：
      • 谁先 CAS 成功，谁生效
