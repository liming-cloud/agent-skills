### 3.3 分帐单生成分布式锁

#### 3.3.1 特性用途

使用redis缓存特性，通过判断插入是否成功避免重复操作导致数据一致性问题

#### 3.3.2 业务说明

单个记录生成分帐单时，通过setIfAbsent方法，避免重复操作

#### 3.3.3 存储设计

|  项/值   |               数值               |                       说明                       |
| :------: | :------------------------------: | :----------------------------------------------: |
|    库    |                0                 |                                                  |
| 数据结构 |              String              |                                                  |
|   ttl    |              30min               |                                                  |
|   key    | billing:‘lock:process:{recordId} | 0018dc4f28724959a92d69520790cbbf标识分帐单记录Id |

**数据格式**

|        字段名         | 字段类型 | 是否可空 |   备注   |
| :-------------------: | :------: | :------: | :------: |
| subContractorRecordId |  String  |    否    | 分表单Id |

#### 3.3.4 预估数据

key数量随操作分帐单记录生成分帐单文件而增加，每个key预估占用空间<1kb,由于设置30min过期，不会长时间占用内存

#### 3.3.5 多团队协同

不涉及多团队协同
